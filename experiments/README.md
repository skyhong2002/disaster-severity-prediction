# Experiment Tracking

Each training run writes a timestamped directory here:

```text
experiments/<run_id>/
├── config.json                 # model family, parameters, feature setup
├── metrics.json                # validation MAE and artifact paths
├── submission_metadata.json    # prediction statistics after predict.py
├── models/                     # ignored by git; large model artifacts
├── submissions/                # ignored by git; generated CSV files
└── logs/                       # ignored by git
```

`experiments/latest.txt` is generated by `src/train.py` and points `src/predict.py`
to the most recent run. Large artifacts are intentionally ignored by git, while
small JSON summaries can be committed when they are useful for the report.

Recommended run naming:

```bash
uv run python src/train.py --experiment-name lgbm_v1
uv run python src/predict.py
uv run python src/predict.py --run-dir experiments/<run_id>
```

When adding a new algorithm, use a distinct `--model-family` and keep its config
and metrics in the same experiment directory format so report tables can be
updated from comparable records.
