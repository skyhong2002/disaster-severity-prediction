\section{Project Summary}

\subsection{Goal}
The goal of this project is to predict the drought severity score for each
geographic region over the \textbf{next five consecutive weeks} (35 days),
given 91 days of historical meteorological observations per region from the
test set. Scores range from 0 (no disaster) to 5 (extreme severity).

\subsection{High-Level Approach}
Our method follows a supervised machine learning paradigm treating the task as
a \textbf{multi-step time series regression} problem. We extract rich temporal
features from the raw meteorological columns including lag values, rolling
means/stds, exponential weighted averages, and calendar features. These
features feed into an ensemble of gradient-boosted tree models. Final
predictions are post-processed to clip values within [0, 5].

\subsection{Related Works}
Time series forecasting for environmental phenomena has been widely studied.
Gradient boosting methods, notably XGBoost~\cite{chen2016xgboost} and
LightGBM~\cite{ke2017lightgbm}, have demonstrated superior performance on
tabular time series data by capturing non-linear feature interactions without
requiring extensive data preprocessing. For weather and disaster prediction,
feature engineering on lag windows has been shown to be highly effective
\cite{kaggle_m5}. More recently, Transformer-based models such as
Temporal Fusion Transformer (TFT)~\cite{lim2021temporal} and PatchTST
\cite{nie2023patchtst} have achieved state-of-the-art results on multivariate
time series benchmarks, which we also explore as a complementary approach.

\section{Proposed Method}

\subsection{Problem Formulation}
Let $\mathcal{R} = \{r_1, r_2, \ldots, r_K\}$ denote the set of geographic
regions. For each region $r_k$, we observe a time series of meteorological
features $\mathbf{X}_{r_k} \in \mathbb{R}^{T \times D}$ where $T$ is the
number of observed days and $D$ is the number of meteorological variables.
The target $y_{r_k, t}$ is the drought severity score at week $t$, available
only on a specific day per week (the rest are NaN).

Given the 91-day test window, the task is to predict:
\begin{equation}
  \hat{y}_{r_k, t+1}, \hat{y}_{r_k, t+2}, \ldots, \hat{y}_{r_k, t+5}
\end{equation}
\noindent minimizing the Mean Absolute Error:
\begin{equation}
  \text{MAE} = \frac{1}{K \cdot 5} \sum_{k=1}^{K} \sum_{w=1}^{5}
  \left| \hat{y}_{r_k, w} - y_{r_k, w} \right|
\end{equation}

\subsection{Data Preprocessing}

\subsubsection{Handling NaN Scores}
Since scores are only recorded weekly, we forward-fill and backward-fill NaN
score values within each 7-day window to produce a complete weekly signal for
feature construction. During training, only the true weekly score rows are
used as labels.

\subsubsection{Feature Engineering}
For each meteorological column $d$, we construct the following feature groups:

\begin{itemize}
  \item \textbf{Lag features}: $x_{t-1}, x_{t-7}, x_{t-14}, x_{t-21},
        x_{t-28}, x_{t-35}$
  \item \textbf{Rolling statistics}: mean, std, min, max over windows of
        7, 14, 21, 28, 35 days
  \item \textbf{EWM}: exponentially weighted mean with $\alpha \in
        \{0.3, 0.1\}$
  \item \textbf{Trend}: linear regression slope over last 7/14 days
  \item \textbf{Calendar}: day of year, week of year, month, quarter
  \item \textbf{Score history}: last 1/2/3/5 weekly scores per region
\end{itemize}

\subsection{Model Architecture}

\subsubsection{LightGBM Ensemble}
We train one LightGBM regressor per prediction horizon (week 1 through week 5),
a technique known as \textbf{Direct Multi-Step Forecasting}. Each model uses:
\begin{itemize}
  \item \texttt{num\_leaves}: 127
  \item \texttt{learning\_rate}: 0.05
  \item \texttt{n\_estimators}: 2000 with early stopping
  \item \texttt{subsample}: 0.8, \texttt{colsample\_bytree}: 0.8
\end{itemize}

\subsubsection{XGBoost Baseline}
We also train an XGBoost model with identical hyperparameters to serve as a
cross-validation reference.

\subsubsection{Temporal Fusion Transformer (TFT)}
% TODO: Add TFT results if used
As a deep learning complement, we experiment with TFT~\cite{lim2021temporal}
using PyTorch Forecasting, encoding each region as a categorical static
covariate.

\subsection{Ensemble Strategy}
Final predictions are produced by weighted averaging:
\begin{equation}
  \hat{y} = \alpha \cdot \hat{y}_{\text{LGB}} + \beta \cdot
  \hat{y}_{\text{XGB}} + \gamma \cdot \hat{y}_{\text{TFT}}
\end{equation}
\noindent where $\alpha, \beta, \gamma$ are tuned via cross-validation.

\subsection{Experimental Setup}
We use a \textbf{time-series cross-validation} strategy with 5 folds, where
each fold's validation set consists of the most recent 5 weeks. All
hyperparameters are tuned on the validation set to prevent data leakage.

\subsection{Baselines}
We define the following self-defined baselines:
\begin{itemize}
  \item \textbf{Baseline-Last}: Predict last observed score for all 5 weeks.
  \item \textbf{Baseline-Mean}: Predict region-wise historical mean score.
  \item \textbf{Baseline-MA7}: 7-week moving average.
\end{itemize}

\section{Conclusion}
In this work, we presented a time series prediction framework for forecasting
drought severity levels from meteorological data. Our proposed pipeline
combining temporal feature engineering with LightGBM achieves competitive
performance on the Kaggle leaderboard. Future work includes exploring
attention-based models and ensemble refinement.
